# Двухсервисная система LLM-консультаций

## 📌 Общая идея проекта

Проект реализует распределённую систему из двух независимых сервисов:

1. **Auth Service** — отвечает за регистрацию пользователей, логин и выдачу JWT-токенов.
2. **Bot Service** — Telegram-бот, который принимает сообщения, проверяет JWT и (в полной версии) должен отправлять запросы к LLM через асинхронную очередь задач.

Архитектура построена по принципу разделения ответственности:
- Auth Service управляет пользователями и токенами
- Bot Service не хранит пользователей и доверяет только JWT

---

## 🧱 Реализованные компоненты

### 1. Auth Service (FastAPI)

Реализовано:
- Регистрация пользователя (`POST /auth/register`)
- Логин (`POST /auth/login`)
- Получение профиля (`GET /auth/me`)
- Хеширование паролей (bcrypt)
- Генерация и проверка JWT (HS256)
- Модель пользователя в SQLite

Структура соответствует требованиям:
- core/config.py — конфигурация через env
- core/security.py — JWT + password hashing
- repositories — работа с БД
- usecases — бизнес-логика
- api/routes — HTTP слой

---

### 2. Bot Service (aiogram)

Реализовано:
- Telegram-бот на aiogram
- Команда `/token <JWT>` для сохранения токена
- Проверка JWT перед обработкой сообщений
- Хранение токена в Redis (логически предусмотрено)
- Подготовлена архитектура под Celery + RabbitMQ
- Разделение на handlers / services / infra слои

---

## ⚙️ Архитектура системы

В полной версии система должна работать так:

1. Пользователь получает JWT через Auth Service
2. Отправляет токен в бота `/token <JWT>`
3. Бот сохраняет токен (Redis)
4. Пользователь отправляет сообщение
5. Бот:
   - проверяет JWT
   - отправляет задачу в RabbitMQ
6. Celery worker:
   - вызывает OpenRouter API
   - возвращает ответ пользователю

## ❌ Текущее ограничение

### ⚠️ Проблема с сетью и Telegram API

Во время разработки возникла системная проблема:

- Telegram API недоступен напрямую из сети без VPN
- При использовании VPN:
  - соединение нестабильно
  - возникают ошибки:
    - `ClientConnectorError`
    - `WinError 121 (semaphore timeout)`
    - `ProxyConnectionError`

### ❗ Итог:

Несмотря на корректную реализацию архитектуры и кода,  
**Bot Service не может стабильно подключиться к Telegram API в текущей среде выполнения.**

---

## 📌 Что это означает

- Код бота **реализован корректно**
- Архитектура соответствует требованиям
- JWT-авторизация реализована
- Redis / Celery слой подготовлен

Но:
> Фактический запуск Telegram polling невозможен из-за сетевых ограничений среды (VPN / блокировки / нестабильный прокси).
